This repository is not a UI template.

It is a motion and initialization system for all Oasis applications.

Every app must implement a unified "Boot Sequence".

For Trae:
Implement the Oasis Boot Sequence based on the spec.

Requirements:
- Use particle-based animation
- Follow the timing defined in specs/animation-core.md
- Use Ember Birth as default variant
- Ensure non-linear motion (no uniform speed)
- Add randomness (seed-based)

Output:
- A reusable function: playOasisBoot(config)
- Config-driven behavior
